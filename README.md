# branchkit-gen

Codegen and validation for [BranchKit](https://branchkit.dev) plugin manifests.

- **Generate** typed action param structs from your `plugin.json`'s `action_types` block (`actions_gen.go` and/or `actions_gen.ts`).
- **Validate** your `plugin.json` against the manifest contract — same checks the BranchKit runtime applies, runnable in CI before you ship.

## Install

```bash
go install github.com/branchkit/branchkit-gen@latest
```

Or clone and build locally:

```bash
git clone https://github.com/branchkit/branchkit-gen.git
cd branchkit-gen
go build -o branchkit-gen .
```

## Usage

### Generate

```bash
# Generate for a single plugin
branchkit-gen --plugin ./my-plugin

# Generate for every subdirectory containing a plugin.json
cd plugins && branchkit-gen --all
```

The tool detects which languages to emit based on the plugin's source directory:
- **Go** output (`src/actions_gen.go`) — emitted when `src/go.mod` exists
- **TypeScript** output (`src/actions_gen.ts`) — emitted when `src/package.json` exists

### Validate

```bash
# Validate the plugin.json in the current directory
branchkit-gen validate

# Validate a specific plugin directory
branchkit-gen validate ./my-plugin

# Validate every subdirectory containing a plugin.json
branchkit-gen validate --all

# JSON output for CI
branchkit-gen validate --json ./my-plugin
```

`validate` checks the same rules the BranchKit runtime applies at plugin load time: `id` and `action_prefix` format, `min_api_version` semver, `settings_tabs` key safety, `dispatch_via` consistency, well-formed `action_types` (valid `field_type`, `enum` declarations carry `enum_values`), and informational notes for unrecognized top-level fields. Errors exit non-zero so CI can gate on a clean run.

## Example

Given this `plugin.json`:

```json
{
  "action_prefix": "music",
  "action_types": {
    "play": {
      "label": "Play Track",
      "fields": [
        { "key": "uri", "field_type": "string" },
        {
          "key": "mode",
          "field_type": "enum",
          "enum_values": ["shuffle", "repeat", "once"],
          "required": true
        }
      ]
    },
    "pause": { "label": "Pause" }
  }
}
```

Running `branchkit-gen --plugin .` produces `src/actions_gen.go`:

```go
// AUTO-GENERATED by emit-sdk — do not edit.
package main

type PauseParams struct {
}

type PlayMode string

const (
	PlayModeShuffle PlayMode = "shuffle"
	PlayModeRepeat  PlayMode = "repeat"
	PlayModeOnce    PlayMode = "once"
)

type PlayParams struct {
	Uri  *string  `json:"uri,omitempty"`
	Mode PlayMode `json:"mode"`
}
```

Your handler imports and uses these types directly:

```go
plugin.HandleAction("music.play", func(req *shared.OnActionRequest) (any, error) {
    var p PlayParams
    req.UnmarshalParams(&p)
    switch p.Mode {
    case PlayModeShuffle:
        // ...
    }
    return nil, nil
})
```

## Supported field types

| `field_type` | Go | TypeScript |
|---|---|---|
| `string` (default) | `string` / `*string` | `string` |
| `int` | `int` / `*int` | `number` |
| `number` | `float64` / `*float64` | `number` |
| `boolean` | `bool` / `*bool` | `boolean` |
| `string[]` | `[]string` | `string[]` |
| `enum` (with `enum_values`) | typed `string` + `const` block | literal union |
| `object` (with nested `fields`) | `json.RawMessage` | `unknown` |
| `json` (escape hatch) | `json.RawMessage` | `unknown` |

Non-required fields are pointer-wrapped in Go and optional (`?`) in TypeScript.
